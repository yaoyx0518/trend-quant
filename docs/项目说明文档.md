# 趋势ETF量化系统项目说明（基于当前代码）

- 文档版本：`v1.1-as-built`
- 更新日期：`2026-03-14`
- 代码基线目录：`E:\codex project\tread quant`
- 运行形态：单用户、本机运行、人工执行交易、系统负责信号与回测分析

## 1. 项目目标与定位

本系统面向 A 股场内 ETF 趋势交易，围绕你的自定义 `Trend Score` 指标构建完整闭环：

1. 行情获取与本地保存（主 `efinance`，备 `akshare`）
2. 日内轮询与 `14:45` 终盘前信号计算
3. 风险预算仓位建议（海龟风险预算思路 + ATR 风险单位）
4. 回测与可视化展示
5. 手工成交回写，保证“建议仓位”和“真实仓位”一致性
6. 全流程日志留痕，支持回溯

V1 不做自动下单，通知渠道先以日志输出为主，后续再接入飞书/邮件实发。

## 2. 当前功能状态（已实现）

## 2.1 数据层

1. 数据源接口抽象：`src/data/provider_base.py`
2. 已接入数据源：
   - `src/data/provider_efinance.py`
   - `src/data/provider_akshare.py`
3. 数据源优先级可配置：`config/app.yaml -> app.data_provider_priority`
4. 能力覆盖：
   - ETF 日线历史（OHLCV + amount）
   - 分钟级历史（30m/60m 等）
   - 最新行情快照（用于盘中信号）
5. 本地行情存储：SQLite（`data/trend_quant.db` 中的 `market_data` 表）

## 2.2 策略与信号

1. 策略实现：`src/strategy/trend_score_strategy.py`
2. 指标计算：`src/strategy/indicators.py`
3. 信号引擎：`src/engine/signal_engine.py`
4. 支持逻辑：
   - `BUY / SELL / HOLD`
   - 进场阈值、离场阈值
   - 硬止损 + 吊灯止损
   - T+1 可卖数量约束（通过 `sellable_qty`）
5. 盘中异常行情保护：实时价格相对昨收跳变超 30% 时忽略

## 2.3 组合与仓位

1. 风险预算头寸计算：`src/portfolio/risk_sizer.py`
2. 计算方式：每标的风险预算 + ATR 风险单位 -> 理论股数
3. 多标的超资金时按比例缩放
4. 100 份整手约束
5. 默认风险预算可按标的配置：
   - `config/instruments.yaml -> risk_budget_pct`
   - `config/instruments.yaml -> stop_atr_mul`

## 2.4 调度任务

调度器：`src/core/scheduler.py`，由 `src/app/main.py` 启动：

1. 盘中轮询任务：`app.polling_times`
2. 收盘前终盘信号：`app.final_signal_time`（当前 `14:45`）
3. 收盘后更新任务：`app.update_time_after_close`（当前 `15:30`）
4. `14:45` 数据不可用重试机制（次数/间隔可配置）

## 2.5 回测能力

核心引擎：`src/backtest/backtest_engine.py`

1. 默认回测起始：`2024-01-01`
2. 回退起始：`2018-01-01`（数据不足时）
3. 初始资金默认：`200000`（可在页面输入覆盖）
4. 回测输出：
   - 策略指标 + 基准指标
   - 净值曲线（策略 vs 基准）
   - 买卖点标记
   - 多标的趋势值曲线
   - 多标的日持仓堆叠柱图（含现金）
   - 回撤曲线
   - 月度热力图（红涨绿跌，深浅反映幅度）
   - 标的维度统计表（交易次数/胜率/盈亏比/贡献）
   - 年度收益表
   - 逐笔交易表（含趋势值、买入仓位占比、卖出盈亏）

## 2.6 前端页面

1. 系统概览：`/`
2. 配置预览：`/config`
3. 回测中心：`/backtest`
4. 手工成交录入：`/trades`
5. 日志页面：`/logs`

回测页已实现图表联动缩放，且支持：

1. 趋势图多选显示指定标的
2. 资产 K 线单标的切换 + 买卖点
3. K 线纵轴随缩放区间动态自适应

## 2.7 存储与日志

1. 数据库：`data/trend_quant.db`（SQLite）
   - `market_data`：ETF 历史行情
   - `signals`：每日信号
   - `signal_states`：最新信号状态
   - `backtests`：回测结果
   - `manual_trades`：手工成交
   - `position_snapshots`：仓位快照
   - `optimization_jobs`：参数优化任务
2. 保留的 JSON 文件：
   - `data/runtime/advice/*.json`（操作建议）
3. 日志：
   - 应用日志：`logs/app/app.log`
   - 计算日志：`logs/calc/calc.jsonl`
4. 日志保留策略：永久保留（配置项 `logging.keep_forever: true`）

## 4. 手工交易回写机制

录入入口：`/trades` 页面 + `POST /trades/api/manual`

录入字段：

1. `trade_date`
2. `symbol`
3. `side` (`BUY`/`SELL`)
4. `qty`
5. `price`
6. `fee`
7. `trade_time`
8. `note`

回放逻辑：

1. 按时间顺序重建仓位（FIFO lot）
2. 计算现金变化
3. 计算可卖数量（T+1）
4. 生成组合快照供信号引擎与概览页使用

## 8. 运行方式（Windows）

## 8.1 环境准备

1. 创建虚拟环境：`python -m venv .venv`
2. 安装依赖：`.\.venv\Scripts\python.exe -m pip install -e .`

## 8.2 启动

1. 直接启动：
   - `.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000 --app-dir src`
2. 若使用 `scripts/run_dev.ps1` 被策略拦截，可临时执行：
   - `powershell -ExecutionPolicy Bypass -File .\scripts\run_dev.ps1`

## 9. 风险与已知限制

1. 交易日历在 `efinance` 不稳定时会退化到“工作日判断”，存在节假日误判风险（`akshare` 可缓解）。
2. 当前为人工执行交易，不是自动下单系统。
3. 当前仅单账户场景，未做多账户隔离。
4. 通知真实送达链路尚未启用（仅日志输出）。
